SQL Identifiers

=================

SQL Identifiers are used to identify database objects such as database names, table names, column names, 

view names, index names, constraints , triggers, stored procedures..etc



While we are declaring identifiers we have to follow some rules and regulations.



1.The only allowed characters to define identifiers a-z, A-Z or 0-9, \_, $

2.must be begin with letter or underscore(\_)

3.we cant declare identifiers with reserved words.

4.spaces are not considered to declare identifiers.

5.identifiers are case sensitive.

6.max length to declare identifiers are 15 characters.



SQL:

======

SQL is a Structured Query Language , it is a Standardized programming language specially designed for 

managing and manipulated relational database. it provides a declarative syntax that allows users to define 

what data they want retrieve, insert, update , or delete without specifying exactly how databases should 

execute these operations.

DDL(Data Definition Language)

==============================

DDL Commands are used to define, create, modify and delete the structure of a database objects

such as databases, tables, views, indexes and constraints.



Create : by using these command to  create a new database , tables, views, indexes..etc.



Alter : by using alter command we and modifies the structure of an existing table. like(Add column, drop column, rename column, modify datatype, add constraints)



Drop : deletes an entire database objects permanently like (databases, tables, view, indexes..etc)



Truncate : by using truncate command delete all records from a table without deleting the table.



Rename : it is used to rename the database object names.



DML(Data Manipulation Language)

================================

Manages data stored within database objects.



Insert : by using insert command we can insert some records into a table.



Update : by using the update command we can update the records based on some condition.



delete : by using the delete command we can delete some records based on condition



lock : controls concurrency by locking a table restrict accesses from other transactions while it is being modified.



DCL(Data Control Language)

===========================

Manages permissions and access control on database objects.



Grant : Gives specific previlages  or permissions(such as select, update, insert, delete, all priviages) to a database user or role



Revoke : removes previously granted permission from database user or role. restricting their access to database objects.



TCL(Transaction control Language)

==================================

manages transactions to maintain data integrity



Commit : permanently saves all changes made during the current transactions to the database. making them visible to other users.



Rollback : undo/reverts changes made during the current transaction back to the last commit state or a save point in case of an error.



Save Point: set a temporary market within a transaction to you which you can later rollback , without undoing the entire transaction.

===========================

used to fetch or retrieve data from the databases.



select : retrieve data from one or more tables or views based on specified columns and conditions. it is the most frequently used command in
